Dynamic SQL

Never write another SQL Query

Rapid development

The majority of database operations are simple lookups (SELECT * Style Queries), most updates are simple updates and additions. handling these automatically allows developers to focus on tasks at hand, not managing data.

Performance and Efficiency

Simplistic writing, efficient pre-processing allows the ease of Select * with the performance of a well tuned query.

Extensibility and Caching

With centralized code for database operations, implementing caching strategies can improve overall program database performance. Other features such as analytics and monitoring can be added seamlessly for all database bound objects.
